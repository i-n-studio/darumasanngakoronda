# Git運用ガイド

本プロジェクトでの Git 運用ルールや、ドキュメント更新、スプリント記録の方法をまとめます。

## 一、ブランチ戦略
- main: リリースを切る安定ブランチ
- develop: 日常開発用にすべての完成済み機能を統合する
- feature/<short-desc>: 1機能1ブランチ
- fix/<ticket> or hotfix/<desc>: 緊急対応

## 二、コミットのベストプラクティス
- コミットは小さくまとまる単位で行う
- コメントは日本語で良いが先頭にタイプをつける（例: feat:, fix:, docs:）
- 例: "docs: 実機テスト手順を README に追加"

## 三、ドキュメントのバージョン管理
- `docs/開発状況.md` は常に最新の進捗がわかるように、
  - スプリント完了後にまとめて記載
  - 臨時の調査/課題は小見出しで追加
- スプリントログは `docs/sprints/` に保存する

## 四、コードレビューとPR
- PRのタイトルは日本語 + 英語の組み合わせでもOK
- 少なくとも1人の承認を得ること
- ドキュメント更新を含む場合は `docs:` プレフィックスを使う

## 五、DEV_STATUS 更新例 (日付: 2025-11-18)
- 進捗: バランスチャレンジの実装、セーフエリアの移動、Hellアンロックの追加
- 変更: `js/game.js` を追加・更新
- 実機テスト: iPhone 12 上でDeviceMotion permission 確認済み

## 六、便利なコマンド
- スプリントのテンプレファイルを作成
  - `./scripts/update_dev_status.sh "スプリント名"`
- 変更の確認
  - `git status` / `git diff` / `git log --oneline`

## 七、追加運用提案
- リリースごとに `CHANGELOG.md` を作成・更新する
- スプリント締めでは `docs/開発状況.md` と `docs/sprints/*.md` に必ず記録
